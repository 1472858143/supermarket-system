# 02 - 商品分类独立化升级

> 记录时间：2026-05-30
> 项目阶段：优化阶段 — 将商品分类从自由文本字段独立为二级分类模块

---

## 一、背景与动机

原 product 表的 `category` 字段为 VARCHAR(50) 自由输入，存在以下问题：

- 无统一分类管理，名称可能拼写不一致
- 无法按分类进行统计分析
- 无分类层级结构

本次升级将分类独立为单独模块，支持严格二级分类体系（一级 → 二级 → 商品）。

---

## 二、设计决策

| 决策点 | 方案 | 理由 |
|---|---|---|
| 分类层级 | 严格二级，商品必须挂在二级分类下 | 超市场景足够，避免过度设计 |
| 排序策略 | 直接存储 sort_order，不做自动重排 | 分类数量极少（< 30），KISS 原则 |
| 查询接口 | 仅一个 GET 返回完整树 | 数据量小，前端内存过滤即可 |
| 单个查询接口 | 不实现 GET /{id} | 前端已有完整树数据，无使用场景（YAGNI） |
| 字段命名 | `create_time` 而非 `created_at` | 与现有表保持一致 |
| 数据迁移 | 开发阶段清空数据重建 | 尚无生产数据 |

---

## 三、数据库变更

### 新增表：category

| 字段 | 类型 | 说明 |
|---|---|---|
| id | BIGINT PK AUTO_INCREMENT | 主键 |
| name | VARCHAR(50) NOT NULL | 分类名称 |
| parent_id | BIGINT NULL | 父分类ID，NULL=一级 |
| sort_order | INT DEFAULT 0 | 排序序号 |
| create_time | DATETIME | 创建时间 |

约束：`UNIQUE(parent_id, name)`，`FK(parent_id) → category(id)`

### 改造表：product

- 删除：`category VARCHAR(50)`
- 新增：`category_id BIGINT NOT NULL`，`FK → category(id)`

迁移脚本：`system/sql/02_add_category_table.sql`

## 四、后端变更

### 新增文件（6 个）

```
category/
├── controller/CategoryController.java   4 个端点：GET树/POST/PUT/DELETE
├── dto/CategoryRequest.java             name(@NotBlank) + parentId + sortOrder
├── entity/Category.java                 POJO，5 个字段
├── mapper/CategoryMapper.java           JdbcTemplate，10 个方法
├── service/CategoryService.java         树组装 + CRUD + 校验逻辑
└── vo/CategoryVO.java                   含 children 列表用于树响应
```

### API 端点

| 方法 | 路径 | 权限 | 功能 |
|---|---|---|---|
| GET | /api/categories | 登录后 | 返回完整分类树 |
| POST | /api/categories | ADMIN | 创建分类（自动生成 sortOrder） |
| PUT | /api/categories/{id} | ADMIN | 更新名称/排序 |
| DELETE | /api/categories/{id} | ADMIN | 删除（有子分类或商品时拒绝） |

### 修改文件（6 个）

| 文件 | 变更内容 |
|---|---|
| Product.java | `category`(String) → `categoryId`(Long) |
| ProductRequest.java | `@NotBlank category` → `@NotNull categoryId` |
| ProductVO.java | `category` → `categoryId` + `categoryName` |
| ProductMapper.java | 所有 SQL 改用 `category_id`，搜索 JOIN category 表 |
| ProductService.java | 注入 CategoryMapper，toVO() 解析分类名称 |
| ReportMapper.java | warningStocks() LEFT JOIN category 获取分类名 |
