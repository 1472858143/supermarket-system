## 五、数据库表结构

数据库名：`market`，字符集 `utf8mb4`，共 9 张表。

| 表名 | 用途 | 关键约束 |
|---|---|---|
| user | 用户账号 | UNIQUE(username) |
| role | 角色 | UNIQUE(role_name), UNIQUE(role_code) |
| user_role | 用户-角色关联 | UNIQUE(user_id, role_id), FK |

| stock | 当前库存 | UNIQUE(product_id), CHECK(quantity >= 0) |
| inbound_order | 入库记录 | FK → product, CHECK(quantity > 0) |
| outbound_order | 出库记录 | FK → product, CHECK(quantity > 0) |
| stock_check | 盘点记录 | FK → product, CHECK(quantities) |
| stock_log | 库存变更日志 | FK → product, CHECK(after_quantity >= 0) |

## 七、已发现的问题与优化方向

### 安全类

| # | 问题 | 严重度 |
|---|---|---|
| 1 | 数据库密码硬编码在 application-dev.yml 并提交到 Git | 高 |
| 2 | JWT 密钥有默认值，未设置环境变量时可被伪造 | 高 |
| 3 | 登录表单预填 admin/admin123 | 中 |
| 4 | CORS 允许所有来源（*） | 低 |
| 5 | 无 HTTPS 强制跳转 | 中 |
| 17 | 登录接口无暴力破解防护（限流） | 中 |
| 18 | 登出仅客户端清除 token，服务端未失效 | 中 |

### 架构 / 代码质量类

| # | 问题 | 严重度 |
|---|---|---|
| 6 | 所有 SQL 内联在 Java 代码中，无 MyBatis XML 或注解 SQL | 中 |
| 7 | Report/System 模块返回 Map<String, Object>，无类型安全 | 中 |
| 8 | 零测试文件（pom.xml 有 test starter 但无测试类） | 高 |
| 10 | 入库/出库商品列表硬编码取前 100 条 | 低 |
| 11 | 入库/出库未校验商品是否上架状态 | 中 |
| 13 | UserService.toVO() 存在 N+1 查询问题 | 中 |
| 14 | stock_log 表无业务索引，报表查询可能慢 | 低 |
| 15 | GlobalExceptionHandler 无 log.error()，服务端错误不可见 | 中 |
| 19 | node_modules 可能被 Git 跟踪（.vite/deps 有变更） | 中 |
| 20 | 前端无 ESLint/Prettier，devDependencies 为空 | 低 |
| 22 | 商品删除未校验是否存在关联业务单据 | 中 |

## 八、优化阶段待办方向（预判）

基于以上问题，优化工作大致可分为以下几个维度：

1. **安全加固** — 密钥管理、登录限流、token 失效机制、CORS 收紧
2. **代码质量** — 引入 MyBatis 替代内联 SQL、消除 N+1 查询、添加单元测试
3. **前端工程化** — 引入 Element Plus、配置 ESLint/Prettier、排除 node_modules
4. **可观测性** — 全局异常日志记录、关键操作审计
5. **数据库优化** — 补充索引、报表查询优化
6. **业务健壮性** — 商品删除前置校验、入库出库状态校验、分页边界处理
