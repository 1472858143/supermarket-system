# 06 - 库存批次功能升级

> 记录时间：2026-06-01
> 项目阶段：优化阶段 - 为采购入库补充库存批次能力，并支持按 SKU 查看批次库存

---

## 一、背景与目标

采购入库模块已经支持一张采购单包含多条 SKU 明细，但库存仍只维护 SKU 总库存，缺少批次维度的信息。实际采购入库需要记录每批商品的数量、进价、生产日期、保质期和到期日期，后续才能支撑效期管理、批次追踪和按批次出库。

本次升级目标：

- 一张采购单仍可包含多个 SKU 明细。
- 当前阶段每条采购明细自动生成一个库存批次。
- 批次绑定 SKU，并保存原始数量、剩余数量、进价、生产日期、保质期、到期日期。
- 批次号由后端自动生成。
- 采购入库创建时同步写入总库存流水和批次流水。
- 库存管理页面支持查看指定 SKU 的批次列表。

---

## 二、核心流程

采购入库创建时，每条明细按以下顺序处理：

```text
写入 purchase_inbound_item
  ↓
创建 stock_batch
  ↓
StockService.increase() 增加 SKU 总库存并写 stock_log
  ↓
StockBatchService.writePurchaseInboundLog() 写 stock_batch_log
```

当前 `stock_batch_log` 由 `StockBatchService` 负责组织写入，实际 SQL 插入由 `StockBatchMapper.insertLog()` 执行。

## 三、数据库变更

### 新增表

| 表名 | 说明 |
|---|---|
| `stock_batch` | 库存批次表，记录 SKU 批次数量、进价、生产日期、保质期、到期日期 |
| `stock_batch_log` | 库存批次流水表，记录批次库存变化 |

### 修改表

| 表名 | 变更内容 |
|---|---|
| `purchase_inbound_item` | 新增复合索引 `idx_pi_item_id_sku (id, sku_id)`，用于批次表复合外键校验 |

### 关键约束

- `stock_batch.purchase_inbound_item_id` 唯一，保证当前阶段一条采购明细只对应一个批次。
- `stock_batch.initial_quantity > 0`。
- `stock_batch.quantity >= 0` 且 `quantity <= initial_quantity`。
- `stock_batch.expire_date >= production_date`。
- `stock_batch_log.before_quantity >= 0`。
- `stock_batch_log.after_quantity >= 0`。
- `stock_batch_log.change_quantity <> 0`。

### 迁移脚本

- `system/sql/08_add_stock_batch.sql`
- `system/sql/00_终版.sql` 已同步补充批次表结构。
- `system/sql/market.sql` 已同步补充批次表结构。

本地 dev 数据库已执行批次迁移，确认 `stock_batch`、`stock_batch_log` 和 `idx_pi_item_id_sku` 已创建。


### 新增接口

| 方法 | 路径 | 权限 | 功能 |
|---|---|---|---|
| GET | `/api/skus/{skuId}/stock-batches` | 登录后 | 查询指定 SKU 的库存批次列表 |

### 采购入库调整

| 文件 | 变更内容 |
|---|---|
| `PurchaseInboundItemRequest.java` | 明细新增 `productionDate`、`shelfLifeDays` |
| `PurchaseInboundItemVO.java` | 详情新增 `batchNo`、`productionDate`、`shelfLifeDays`、`expireDate` |
| `PurchaseInboundMapper.java` | 明细插入返回生成 ID；详情查询关联 `stock_batch` |
| `PurchaseInboundService.java` | 每条明细创建批次，并在同一事务内增加库存、写批次流水 |

### 批次业务规则

| 规则 | 说明 |
|---|---|
| 批次号 | `SB` + 日期 + 三位递增序号 |
| 到期日期 | `productionDate.plusDays(shelfLifeDays)` |
| 初始数量 | 使用采购明细换算后的 `baseQuantity` |
| 剩余数量 | 创建时等于初始数量 |
| 入库批次流水 | `before_quantity = 0`，`change_quantity = baseQuantity`，`after_quantity = baseQuantity` |
| 来源类型 | `PURCHASE_INBOUND_ITEM` |
| 变更类型 | `PURCHASE_INBOUND` |

## 五、前端变更

### 采购入库页面

`PurchaseInboundsView.vue` 新增：

- 每条明细填写生产日期。
- 每条明细填写保质期天数。
- 前端预览到期日期。
- 提交采购入库时携带 `productionDate`、`shelfLifeDays`。
- 采购入库详情展示批次号、生产日期、保质期和到期日期。

### 库存管理页面

新增 `system/frontend/src/api/stockBatch.js`：

```js
import request from './request'

export function listStockBatches(skuId) {
  return request.get(`/skus/${skuId}/stock-batches`)
}
```

`StocksView.vue` 新增：

- 库存表操作列常驻。
- 所有库存页用户可点击“批次”按钮。
- “维护上下限”仍仅 ADMIN 可见。
- 新增 SKU 批次弹窗。
- 批次列表展示批次号、批次数量、剩余数量、进价、生产日期、保质期、到期日期、创建时间。
- 批次加载增加请求 token，避免快速切换 SKU 时旧请求覆盖新弹窗数据。

## 九、后续可扩展方向

- 按批次出库，支持先进先出或临期优先。
- 批次库存扣减时写出库型 `stock_batch_log`。
- 库存盘点时同步调整批次数量。
- 临期商品提醒和过期批次筛选。
