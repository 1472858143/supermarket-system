# 11 - 品牌管理模块新增

> 记录时间：2026-06-02
> 项目阶段：优化阶段 - 新增轻量品牌主数据，并将品牌绑定到 SPU 商品主档
> 关联提交：`39bcb1f feat(product): add brand management`

---

## 一、背景与目标

商品资料在前序阶段已经完成分类独立化、SKU 规格化和单位换算升级。随着商品主档能力逐步完善，系统需要补充品牌维度，用于商品检索、展示和后续报表分析。

本次升级采用轻量化方案，不引入复杂品牌体系，只解决当前最核心的主数据管理问题：

- 系统中有哪些品牌；
- 品牌是否启用；
- 商品 SPU 属于哪个品牌；
- 商品列表和维护页面可以按品牌展示、筛选和维护。

本次明确不将品牌直接绑定到 SKU。品牌归属于商品主档 SPU，SKU 继续表达规格、条码、单位和价格等明细属性。

---

## 二、设计决策

| 决策点 | 方案 | 理由 |
|---|---|---|
| 模块定位 | 品牌作为独立主数据模块 | 便于统一维护，也便于后续扩展品牌报表、授权、展示配置 |
| 绑定关系 | `brand 1 - N product` | 品牌属于商品主档维度，避免 SKU 重复维护品牌 |
| SKU 关系 | SKU 不直接增加品牌字段 | 保持 SKU 聚焦规格和计量，不扩大明细表职责 |
| 编码方式 | 后端自动生成 `BRD000001` 格式编码 | 降低手工录入成本，保持编码统一 |
| 启用状态 | `status` 使用 1/0 | 与商品、SKU、供应商等模块保持一致 |
| 删除策略 | 被商品引用的品牌禁止删除 | 避免商品主档出现无效品牌引用 |
| 历史数据兼容 | 迁移时创建默认品牌并回填既有商品 | 支持 `product.brand_id` 设为非空外键 |

## 三、数据库变更

### 新增表：`brand`

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | BIGINT PK AUTO_INCREMENT | 主键 |
| `brand_code` | VARCHAR(50) NOT NULL UNIQUE | 品牌编码 |
| `brand_name` | VARCHAR(100) NOT NULL UNIQUE | 品牌名称 |
| `status` | TINYINT NOT NULL DEFAULT 1 | 0 禁用，1 启用 |
| `remark` | VARCHAR(200) | 备注 |
| `create_time` | DATETIME | 创建时间 |

### 商品表调整

`product` 表新增字段：

```sql
brand_id BIGINT NOT NULL COMMENT '品牌ID，关联brand表'
```

并补充索引和外键：

```sql
KEY idx_product_brand (brand_id),
CONSTRAINT fk_product_brand FOREIGN KEY (brand_id) REFERENCES brand(id)
```

### 迁移脚本

新增迁移脚本：

```text
system/sql/13_add_brand.sql
```

脚本处理内容：

- 创建 `brand` 表；
- 插入默认品牌 `BRD000001 / 默认品牌`；
- 为 `product` 增加 `brand_id`；
- 将历史商品回填到默认品牌；
- 将 `brand_id` 改为非空；
- 增加品牌索引和外键。

新库初始化脚本同步更新：

- `system/sql/market.sql`
- `system/sql/00_终版.sql`

### 新增 API 端点

| 方法 | 路径 | 权限 | 功能 |
|---|---|---|---|
| GET | `/api/brands` | 登录后 | 品牌分页列表，支持关键字和状态筛选 |
| GET | `/api/brands/options` | 登录后 | 启用品牌选项，用于商品表单 |
| POST | `/api/brands` | ADMIN | 新增品牌 |
| PUT | `/api/brands/{id}` | ADMIN | 编辑品牌 |
| DELETE | `/api/brands/{id}` | ADMIN | 删除品牌 |

### 后端业务规则

| 规则 | 说明 |
|---|---|
| 品牌名称必填 | 空名称拒绝保存 |
| 品牌编码自动生成 | 读取当前最大 `BRD%` 编码后递增 |
| 品牌名称唯一 | 新增和编辑时均校验重复 |
| 状态值校验 | 仅允许 0 或 1 |
| 启用品牌校验 | 商品创建和编辑时必须绑定启用品牌 |
| 引用保护 | 已被商品引用的品牌禁止删除 |

### 商品模块调整

商品 SPU 增加品牌字段：

- `Product.brandId`
- `ProductRequest.brandId`
- `ProductVO.brandId`
- `ProductVO.brandCode`
- `ProductVO.brandName`

商品列表接口新增 `brandId` 查询参数，并在关键字搜索中纳入品牌编码和品牌名称。

## 八、影响与注意事项

### 数据库迁移

已有数据库升级时执行：

```text
system/sql/13_add_brand.sql
```

该脚本会为历史商品回填默认品牌，确保 `product.brand_id` 可以安全改为非空。

### 接口兼容

商品新增和编辑请求新增必填字段：

```json
{
  "brandId": 1
}
```

如果旧前端或外部调用方未传 `brandId`，后端会按参数校验拒绝保存。

### 商品模型边界

品牌只绑定 SPU，不绑定 SKU。后续 SKU 查询需要展示品牌时，应通过 `sku.product_id -> product.brand_id -> brand` 关联获取，不应在 SKU 表重复增加品牌字段。
