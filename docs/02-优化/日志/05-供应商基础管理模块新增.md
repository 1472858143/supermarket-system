# 05 - 供应商基础管理模块新增

> 记录时间：2026-06-01
> 项目阶段：优化阶段 — 将供应商作为独立主数据模块补齐

---

## 一、背景与目标

前一阶段新增采购入库模块时，供应商暂未纳入范围。本次优化将供应商从“后续扩展项”落地为独立基础资料模块，当前阶段只解决以下问题：

- 系统中有哪些供应商
- 供应商基本信息如何维护
- 供应商是否启用

本阶段不扩展供应商结算、资质管理、采购订单绑定、供应商绩效等复杂业务，保持主数据能力清晰可控。

---

## 二、设计决策

| 决策点 | 方案 | 理由 |
|---|---|---|
| 供应商定位 | 独立主数据表 `supplier` | 便于后续采购、结算等模块复用 |
| 供应商编码 | 后端自动生成，格式 `SUP000001` | 减少人工录入错误，编码统一 |
| 启用状态 | `status` 保存 1/0 | 与现有商品、SKU 状态模型保持一致 |
| 基本信息 | 名称、联系人、联系电话、地址、备注 | 满足当前基础维护范围 |
| 权限控制 | 登录后可查询，ADMIN 可维护 | 沿用现有基础资料权限模式 |
| 电话格式 | 只允许数字、空格、`+`、`-` | 防止中文、字母等无效联系电话入库 |
| 删除策略 | 当前支持删除 | 后续若被采购单引用，再补充引用校验或改为禁用 |

## 三、数据库变更

### 新增表：supplier

| 字段 | 类型 | 说明 |
|---|---|---|
| id | BIGINT PK AUTO_INCREMENT | 主键 |
| supplier_code | VARCHAR(50) NOT NULL UNIQUE | 供应商编码 |
| supplier_name | VARCHAR(100) NOT NULL | 供应商名称 |
| contact_person | VARCHAR(50) | 联系人 |
| contact_phone | VARCHAR(30) | 联系电话 |
| address | VARCHAR(200) | 地址 |
| remark | VARCHAR(200) | 备注 |
| status | TINYINT DEFAULT 1 | 状态：1 启用，0 禁用 |
| create_time | DATETIME | 创建时间 |

## 四、后端变更

### 新增文件

```
supplier/
├── controller/SupplierController.java   5 个端点：列表/详情/新增/编辑/删除
├── dto/SupplierRequest.java             供应商维护请求
├── entity/Supplier.java                 供应商实体
├── mapper/SupplierMapper.java           JdbcTemplate 数据访问
├── service/SupplierService.java         编码生成、状态校验、电话校验、CRUD
└── vo/SupplierVO.java                   供应商响应视图
```

### 新增 API 端点

| 方法 | 路径 | 权限 | 功能 |
|---|---|---|---|
| GET | /api/suppliers | 登录后 | 供应商分页列表，支持 keyword 搜索 |
| GET | /api/suppliers/{id} | 登录后 | 供应商详情 |
| POST | /api/suppliers | ADMIN | 新增供应商 |
| PUT | /api/suppliers/{id} | ADMIN | 编辑供应商 |
| DELETE | /api/suppliers/{id} | ADMIN | 删除供应商 |

### 后端业务规则

| 规则 | 说明 |
|---|---|
| 供应商名称必填 | 空名称拒绝保存 |
| 供应商编码自动生成 | 读取当前最大 `SUP%` 编码后递增 |
| 编码异常保护 | 当前最大编码不符合 `SUP` + 6 位数字时拒绝继续生成 |
| 状态默认启用 | 未传 status 时默认 1 |
| 状态值校验 | 仅允许 0 或 1 |
| 联系电话校验 | 非空时只允许数字、空格、`+`、`-` |
