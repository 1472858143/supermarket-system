## 3. 当前项目的最终定位

根据当前所有文档，项目的最终定位如下：

**本项目是一个面向超市场景的库存管理系统毕业设计，采用前后端分离架构，以库存模块为核心，通过商品、库存、入库、出库、盘点、报表与系统管理等模块协同完成库存业务闭环。**

该系统不是采购系统，也不是 ERP 全流程系统，而是以库存管理为中心的业务系统。

## 5. 当前核心设计思想

本项目最核心的设计思想如下：

### 5.1 统一库存控制

- `stock` 模块是唯一允许直接修改库存表的模块
- 所有库存数量变化都必须通过 `stock` 模块完成
- 这样可以避免库存数据被多处直接修改而导致不一致

### 5.2 原因模块与状态模块解耦

- `stock` 模块负责维护“库存当前状态”
- `inbound` 模块负责记录“为什么库存增加”
- `outbound` 模块负责记录“为什么库存减少”
- `stockcheck` 模块负责记录“库存差异及调整原因”

### 5.3 报表只读

- `report` 模块只负责查询、统计和展示
- `report` 模块不得修改任何业务数据

### 5.4 毕设导向的轻量化设计

- 结构要清晰，但不需要过度复杂
- `stock` 模块适当强化领域设计
- 其他模块以常规分层实现为主
- `system` 模块保持轻量化，不做复杂配置中心

## 6. 当前功能需求基线

### 6.1 auth 模块

主要功能如下：

- 用户登录
- 用户登出
- 基于 Token 的身份认证
- 接口访问认证

主要依赖数据表如下：

- `user`
- `role`
- `user_role`

### 6.2 user 模块

主要功能如下：

- 用户新增、修改、删除、查询
- 用户启用与禁用
- 用户角色分配

主要依赖数据表如下：

- `user`
- `role`
- `user_role`

### 6.3 product 模块

主要功能如下：

- 商品新增、修改、删除、查询
- 商品上下架管理

主要依赖数据表如下：

- `product`

### 6.4 stock 模块

主要功能如下：

- 库存查询
- 库存上下限维护
- 库存合法性校验
- 库存变更控制
- 库存日志记录

主要依赖数据表如下：

- `stock`
- `stock_log`

关键规则如下：

- 只有该模块可直接修改库存表
- 入库、出库、盘点必须调用该模块

### 6.5 inbound 模块

主要功能如下：

- 新增入库记录
- 查询入库记录
- 记录商品、数量、操作人、时间
- 调用库存模块增加库存

主要依赖数据表如下：

- `inbound_order`
- `stock`
- `stock_log`

### 6.6 outbound 模块

主要功能如下：

- 新增出库记录
- 查询出库记录
- 记录商品、数量、操作人、时间
- 调用库存模块减少库存

主要依赖数据表如下：

- `outbound_order`
- `stock`
- `stock_log`

关键规则如下：

- 库存不足时，出库必须失败

### 6.7 stockcheck 模块

主要功能如下：

- 记录盘点结果
- 计算库存差异
- 保存盘点记录
- 调用库存模块执行库存调整

主要依赖数据表如下：

- `stock_check`
- `stock`
- `stock_log`

### 6.8 report 模块

主要功能如下：

- 库存统计
- 入库统计
- 出库统计
- 库存变化趋势展示
- 库存预警统计

主要依赖数据表如下：

- `product`
- `stock`
- `inbound_order`
- `outbound_order`
- `stock_check`

### 6.9 system 模块

当前 system 模块的定位已经发生调整。

当前功能如下：

- 展示系统基础信息
- 提供系统运行说明
- 提供辅助管理入口

当前约束如下：

- 由于数据库中没有 `system_config`、`operation_log` 等独立表
- 因此当前 `system` 模块不应设计为复杂系统配置中心
- 更适合做轻量化系统信息与辅助管理模块

## 9. 当前数据库设计基线

### 9.1 正式数据库来源

正式数据库脚本为：

- [market.sql](E:\InventoryManagementSystem\Document\3-项目开发\market.sql)

数据库名称如下：

- `supermarket_inventory`

### 9.2 当前核心数据表

当前正式数据表共 9 张：

- `user`
- `role`
- `user_role`
- `product`
- `stock`
- `inbound_order`
- `outbound_order`
- `stock_check`
- `stock_log`

### 9.3 当前数据库关系特征

- 用户与角色：多对多
- 商品与库存：一对一
- 商品与入库记录：一对多
- 商品与出库记录：一对多
- 商品与盘点记录：一对多
- 商品与库存日志：一对多

### 9.4 当前数据库约束重点

依据 `market.sql`，当前数据库已明确以下约束：

- 用户名唯一
- 角色名称唯一
- 角色编码唯一
- 用户角色组合唯一
- 商品编号唯一
- 商品售价不得低于进价
- 库存数量不能小于 0
- 库存上下限必须合法
- 入库数量与出库数量必须大于 0
- 盘点库存不能为负
- 库存日志中的变化后库存不能为负

### 9.5 当前数据库设计结论

当前数据库已经能够支持毕业设计范围内的核心功能实现，因此后续设计与实现应以现有库表为准，不再额外扩展新的核心业务表，除非后续确有必要并经过统一修订。
