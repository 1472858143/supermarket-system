# 数据库设计

## 正式脚本

正式数据库脚本为 [00_终版.sql](../../system/sql/00_终版.sql)，数据库名为 `supermarket_inventory`，字符集为 `utf8mb4`。

所有表结构、字段、约束和外键关系均以该脚本为准。

## 数据表清单

| 表 | 说明 |
|---|---|
| `user` | 后台用户，保存用户名、密码哈希、姓名、状态 |
| `role` | 角色字典，当前固定为 `ADMIN`、`USER` |
| `user_role` | 用户与角色多对多关系 |
| `product` | 商品基础资料 |
| `stock` | 当前库存状态 |
| `inbound_order` | 入库记录 |
| `outbound_order` | 出库记录 |
| `stock_check` | 盘点记录 |
| `stock_log` | 库存变化日志 |

## 关系说明

- 用户与角色：多对多。
- 商品与库存：一对一。
- 商品与入库记录：一对多。
- 商品与出库记录：一对多。
- 商品与盘点记录：一对多。
- 商品与库存日志：一对多。

## 关键约束

- `user.username` 唯一。
- `role.role_code` 唯一。
- `product.product_code` 唯一。
- 商品售价不得低于进价。
- 库存数量不得小于 0。
- 库存上限不得小于库存下限。
- 入库数量和出库数量必须大于 0。
- 盘点系统库存和实际库存不得小于 0。
- 库存日志的变化后库存不得小于 0。

## 库存一致性

`stock` 表保存当前库存状态，`stock_log` 保存库存变化事实。入库、出库和盘点不直接更新 `stock` 表，统一通过后端 `StockService` 执行并写入日志。

这种设计避免库存数量被多个业务模块分散修改，也便于后续通过 `stock_log` 追踪库存变化来源。
