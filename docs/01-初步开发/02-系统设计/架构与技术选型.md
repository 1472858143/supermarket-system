# 架构与技术选型

## 系统定位

本系统是面向超市场景的库存管理毕业设计项目，采用前后端分离架构，围绕商品、库存、入库、出库、盘点、报表和系统信息形成库存业务闭环。

系统不纳入采购、供应商、多仓库、POS、财务对账和复杂审批流。

## 总体架构

```text
浏览器
-> Vue 3 管理后台
-> Axios 请求封装
-> Spring Boot REST API
-> Service 业务编排
-> Mapper 数据访问
-> MySQL
```

部署时由 Nginx 提供前端静态资源，并通过 `/api` 反向代理到后端服务。

## 技术选型

| 层次 | 方案 |
|---|---|
| 前端框架 | Vue 3 |
| 构建工具 | Vite |
| 状态管理 | Pinia |
| 路由 | Vue Router |
| HTTP 客户端 | Axios |
| 后端框架 | Spring Boot 3 |
| 数据访问 | Spring JDBC |
| 认证 | 自定义 JWT + BCrypt |
| 数据库 | MySQL 8 |
| 部署 | Nginx + Spring Boot Jar + MySQL |

## 后端结构

后端按业务模块拆分，模块内部采用 Controller、Service、Mapper、DTO、VO、Entity 分层。

```text
com.supermarket.inventory
├─ auth
├─ user
├─ product
├─ stock
├─ inbound
├─ outbound
├─ stockcheck
├─ report
├─ system
├─ common
└─ config
```

`stock` 模块保留 `domain` 层，用于集中库存增加、扣减、盘点调整和上下限校验。

## 前端结构

前端按页面、接口、组件、布局、状态和工具拆分。

```text
frontend/src
├─ api
├─ assets
├─ components
├─ layout
├─ router
├─ stores
├─ utils
└─ views
```

`views` 与后端业务模块保持一一对应，便于接口联调和文档维护。

## 核心设计原则

- `stock` 是唯一允许直接修改库存表的模块。
- `inbound`、`outbound`、`stockcheck` 只记录库存变化原因，必须调用 `StockService` 完成库存变更。
- `report` 只读，不写业务数据。
- `system` 保持轻量化，不虚构数据库中不存在的配置中心或审计中心。
- 前端只负责展示、输入和调用接口，不计算最终库存。
