# 前端模块设计

## 总体说明

前端采用 Vue 3 + Vite + Pinia + Vue Router + Axios，实现后台管理界面。页面只负责展示、输入、调用接口和展示反馈，不实现库存计算或后端业务规则。

## 目录职责

| 目录 | 职责 |
|---|---|
| `src/api` | 按模块封装接口请求 |
| `src/assets` | 全局样式 |
| `src/components` | 公共表格、弹窗、工具栏、状态标签、权限按钮 |
| `src/layout` | 后台主布局、侧边栏、顶部栏 |
| `src/router` | 路由配置和路由权限 |
| `src/stores` | 登录态、用户信息、角色状态 |
| `src/utils` | Token 和权限工具 |
| `src/views` | 业务页面 |

## 页面设计

| 页面 | 路由 | 说明 |
|---|---|---|
| 登录 | `/login` | 提交账号密码，保存 Token 和用户角色 |
| 首页概览 | `/dashboard` | 展示库存核心指标和预警列表 |
| 用户管理 | `/users` | 管理员维护用户、状态和角色 |
| 商品管理 | `/products` | 商品查询，管理员维护商品资料 |
| 库存管理 | `/stocks` | 查看库存状态，管理员维护上下限 |
| 入库管理 | `/inbounds` | 查询和新增入库记录 |
| 出库管理 | `/outbounds` | 查询和新增出库记录 |
| 盘点管理 | `/stockchecks` | 查询盘点记录，管理员新增盘点 |
| 报表统计 | `/reports` | 只读展示库存变化趋势和汇总数据 |
| 系统信息 | `/system` | 展示轻量系统运行信息 |

## 权限展示

前端保留三层权限展示：

- 路由：`meta.roles` 控制页面访问。
- 菜单：侧边栏按当前角色过滤。
- 按钮：`PermissionButton` 控制写操作按钮显示。

当普通用户没有行级操作时，公共表格不展示空的“操作”列，避免页面出现无意义空列。

## 请求封装

统一请求工具位于 `src/api/request.js`。

- `baseURL` 使用 `VITE_API_BASE_URL || '/api'`。
- 请求自动注入 `Authorization: Bearer <token>`。
- 成功响应返回 `data`。
- `code != 0` 时抛出错误并展示后端 `message`。
- `401` 时清理本地 Token 和用户信息并跳转登录页。

页面中不得直接拼接后端接口地址。

## UI/UX 规范

- 后台采用 Sidebar + Topbar + Content 布局。
- 查询、重置和主要操作集中在工具栏。
- 列表页面统一使用公共表格、分页、loading 和 empty 状态。
- 写操作统一使用弹窗表单，提交中禁用按钮，失败时保留用户输入。
- 状态字段使用标签强化识别，例如启用、禁用、上架、下架、低库存、超上限。
- 移动端下侧边栏转为横向菜单，表格横向滚动，表单改为单列。

## 页面边界

- 商品页面不维护库存数量。
- 库存页面不提供直接修改当前库存数量入口。
- 入库、出库、盘点只调用各自业务接口。
- 报表页面只读，不提供新增、修改、删除或库存调整入口。
- 系统信息页面不展示数据库密码、JWT 密钥等敏感信息。
