# 部署运行说明

## 本地运行

### 后端

```powershell
cd E:\Supermarket\backend
mvn spring-boot:run
```

### 前端

```powershell
cd E:\Supermarket\frontend
npm install
npm run dev
```

开发环境中，前端默认通过 Vite proxy 将 `/api` 转发到 `http://localhost:8080`。

## 数据库初始化

在 MySQL 中执行：

```sql
source E:/Supermarket/system/sql/00_终版.sql;
```

后端启动时会根据配置补齐基础角色和演示账号。默认账号如下：

| 用户名 | 密码 | 角色 |
|---|---|---|
| `admin` | `admin123` | `ADMIN` |
| `user` | `user123` | `USER` |

密码使用 BCrypt 哈希保存，不存储明文。

## 常用环境变量

| 变量 | 说明 |
|---|---|
| `SPRING_PROFILES_ACTIVE` | Spring Profile，如 `dev`、`prod` |
| `DB_URL` | MySQL JDBC 地址 |
| `DB_USERNAME` | 数据库用户名 |
| `DB_PASSWORD` | 数据库密码 |
| `JWT_SECRET` | JWT 签名密钥 |
| `BOOTSTRAP_ENABLED` | 是否初始化默认角色和账号 |
| `VITE_API_BASE_URL` | 前端 API 前缀，默认 `/api` |

## 构建

后端构建：

```powershell
cd E:\Supermarket\backend
mvn test
```

前端构建：

```powershell
cd E:\Supermarket\frontend
npm run build
```

## 部署建议

- 使用 Nginx 托管 `frontend/dist`。
- Nginx 将 `/api` 反向代理到 Spring Boot 后端。
- Spring Boot 后端以 Jar 方式运行。
- MySQL 使用 `00_终版.sql` 初始化。
- 生产环境数据库连接、JWT 密钥和启动账号密码通过环境变量提供。

## 验收要点

- 登录、退出和刷新登录态正常。
- `ADMIN` 可访问全部页面和写操作。
- `USER` 不显示用户管理、商品维护、库存上下限、盘点新增等入口。
- 入库后库存增加，出库后库存减少。
- 库存不足时出库失败。
- 盘点后库存调整为实际库存。
- 报表和系统信息页面只读。
