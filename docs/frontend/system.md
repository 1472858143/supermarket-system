# 系统信息前端开发记录

## 1. 页面定位

系统信息页面对应后端 `system` 模块，用于展示轻量系统信息。页面只读，不提供配置修改、审计查询或运维控制功能。

## 2. 涉及源码

| 类型 | 源码 |
|---|---|
| View | `frontend/src/views/system/SystemView.vue` |
| API | `frontend/src/api/system.js` |
| 路由 | `frontend/src/router/index.js` |

## 3. 页面功能

- 获取系统基础信息。
- 展示系统名称、版本或运行说明类字段。
- 提供只读信息展示。

## 4. 权限控制

- 路由 `/system` 允许 `ADMIN` 和 `USER`。
- 页面无写操作按钮。
- 后端 `/api/system/info` 只要求登录。

## 5. 接口调用

| 前端方法 | 后端接口 | 用途 |
|---|---|---|
| `getSystemInfo()` | `GET /api/system/info` | 获取系统信息 |

## 6. UI/UX 设计

- 使用信息卡片展示系统基础信息。
- 加载中显示 loading。
- 数据为空时显示 empty 或默认说明。
- 不提供配置编辑入口。

## 7. 异常处理

- 接口失败时展示错误提示。
- 401 由请求封装跳转登录。
- 响应字段缺失时页面保持可读，不出现脚本错误。

## 8. 测试与验收

- `ADMIN` 可访问系统信息页。
- `USER` 可访问系统信息页。
- 页面不展示数据库密码、JWT 密钥等敏感信息。
- 页面不提供系统配置修改按钮。
